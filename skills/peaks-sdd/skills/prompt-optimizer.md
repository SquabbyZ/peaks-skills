# Prompt Optimizer - 提示词优化器

帮助用户优化提示词，使得用户的实际表达和 agent 的理解更贴合。

## 工作流程

### Phase 1: 分析原始 Prompt

1. **读取用户原始输入**
2. **识别意图**：功能请求、Bug 修复、架构决策、代码审查等
3. **提取关键信息**：技术栈、约束条件、期望输出

### Phase 2: 检测问题

检测以下常见问题：

| 问题类型 | 描述 | 严重级别 |
|---------|------|---------|
| **模糊性** | 使用"大"、"小"、"快"等模糊词 | HIGH |
| **缺失信息** | 缺少技术栈、上下文、约束 | HIGH |
| **矛盾性** | 前后描述矛盾 | CRITICAL |
| **假设过多** | 对 agent 能力假设过多 | MEDIUM |
| **范围过大** | 一个 prompt 包含太多任务 | HIGH |
| **缺少格式** | 没有指定输出格式 | MEDIUM |

### Phase 3: 生成优化版本

将原始 prompt 转换为结构化版本：

```markdown
## 原始 Prompt
[用户输入]

## 优化后 Prompt
[优化后的版本]

## 优化说明
1. [具体改动和原因]
2. [具体改动和原因]
3. [具体改动和原因]
```

## 优化技术

### 1. 明确意图 (Explicit Intent)

```
# 原始
"帮我看看这个 bug"

# 优化
"帮我分析登录按钮点击无反应的问题：
- 技术栈: React 18 + TypeScript
- 复现步骤: 点击登录按钮后无任何响应
- 已尝试: 清除缓存、重启服务
- 预期: 按钮点击后应显示 loading 并跳转"
```

### 2. 指定输出格式 (Output Format)

```
# 原始
"写个排序算法"

# 优化
"实现一个排序算法：
- 语言: TypeScript
- 算法: 快速排序
- 输出格式:
  ```typescript
  function quickSort<T>(arr: T[]): T[]
  ```
- 要求: 使用泛型、包含单元测试"
```

### 3. 提供上下文 (Context)

```
# 原始
"添加用户注册功能"

# 优化
"添加用户注册功能：
- 上下文: 已有的 auth 模块使用 JWT，需集成到现有系统
- 技术栈: NestJS + Prisma + PostgreSQL
- 要求:
  - 字段: email, password, name
  - 验证: email 格式、密码强度 >= 8 位
  - 已有 User 模型在 schema.prisma 中
- 约束: 需要兼容现有的 /api/auth/login 接口"
```

### 4. 分解复杂任务 (Task Decomposition)

```
# 原始
"实现一个完整的电商系统"

# 优化
"实现电商系统的核心模块：
阶段 1: 用户模块 (本周)
  - 注册/登录
  - 密码重置
阶段 2: 商品模块 (下周)
  - 商品 CRUD
  - 分类管理
阶段 3: 订单模块 (第三周)
  - 下单流程
  - 支付集成

请先实现阶段 1"
```

### 5. 添加约束 (Constraints)

```
# 原始
"写一个 React 组件"

# 优化
"写一个用户头像组件：
- 框架: React 18 + TypeScript
- 样式: Tailwind CSS (禁止内联 style)
- 约束:
  - 组件不超过 100 行
  - 使用组件库 (已有 antd)
  - 支持 fallback (无头像时显示首字母)
- 输出: UserAvatar.tsx + UserAvatar.test.tsx"
```

### 6. 指定验收标准 (Acceptance Criteria)

```
# 原始
"实现搜索功能"

# 优化
"实现搜索功能：
- 验收标准:
  - [ ] 输入关键词能返回匹配结果
  - [ ] 无结果时显示"未找到"提示
  - [ ] 搜索中显示 loading 状态
  - [ ] 响应时间 < 500ms
- 性能要求: 10k+ 商品数据下仍流畅
- 错误处理: 网络失败时显示重试按钮"
```

## 常见优化案例

### Bug 修复 Prompt

```
原始: "登录失败了"
优化:
"登录失败问题排查：
- 环境: 生产环境
- 技术栈: Next.js 14 + Auth.js
- 错误现象: 输入正确密码后显示'登录失败'，无具体错误信息
- 已尝试: 清除 cookie、刷新页面、重置密码
- 预期: 登录成功并跳转到 /dashboard
- 相关日志: [粘贴 relevant log]"

关键优化点:
1. 明确技术栈版本
2. 提供错误现象细节
3. 说明已尝试的解决方式
4. 指定预期行为
5. 附上相关日志
```

### 功能实现 Prompt

```
原始: "添加搜索功能"
优化:
"添加商品搜索功能：
- 技术栈: React 18 + TypeScript + React Query
- 数据源: /api/products (已有分页 API)
- 需求:
  - 关键词搜索 (title, description)
  - 实时搜索 (debounce 300ms)
  - 显示搜索结果数量
  - 空结果处理
- 约束: 使用现有 UI 组件库，不自己实现
- 验收: 10 个测试用例覆盖正常/边界/异常

请先提供技术方案，确认后再实现"
```

### 代码审查 Prompt

```
原始: "看看这个代码有什么问题"
优化:
"代码审查请求：UserService.ts
- 审查重点: 安全性、性能、可维护性
- 技术栈: NestJS + TypeORM
- 相关文件: auth.service.ts (依赖)
- 已知的性能问题: 高并发下响应慢

请按以下顺序审查:
1. 安全漏洞 (SQL 注入、XSS、认证绕过)
2. 性能问题 (N+1 查询、缺少索引)
3. 代码质量 (命名、可读性、测试覆盖)
4. 架构问题 (耦合度、错误处理)"
```
