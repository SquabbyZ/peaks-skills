# 产出物格式模板

## PRD

```markdown
# PRD - [功能名]

## 概述

### 背景
[为什么需要这个功能]

### 目标
[要达成什么]

## 功能列表

### [NEW] [功能点名称]
- 描述：[功能的具体描述]
- 验收标准：
  - [可测试条件 1]
  - [可测试条件 2]

### [CHANGED] [已有功能点名称]
- 原：[原来的行为]
- 新：[新的行为]

### [DEPRECATED] [废弃功能点名称]
- 原因：[为什么废弃]

## 非功能性需求
- 性能：[响应时间/吞吐量]
- 安全：[认证、授权、数据保护]
- 兼容性：[平台/浏览器/版本]
```

## Plan

```markdown
# Plan - [功能名] - YYYYMMDD

## 技术方案
- 前端：[技术栈]
- 后端：[技术栈]
- 数据库：[数据模型/迁移]

## 里程碑
| # | 里程碑 | 依赖 | 验证 |
| --- | --- | --- | --- |
| M1 | [描述] | 无 | [命令/检查] |

## 风险
- [风险] → [应对策略]
```

## Bug 报告

```markdown
# Bug 分析报告 - [问题描述] - YYYYMMDD

## 问题概述
[一句话描述]

## 复现步骤
1. [步骤 1]
2. [步骤 2]

## 预期行为
[期望]

## 实际行为
[实际]

## 根因分析
[证据驱动的根因]

## 修复方案
[最小修复方案]

## 验证
[测试/命令/截图]
```

## 测试报告

```markdown
# 测试报告 - [功能名] - YYYYMMDD

## 测试概览
- 测试时间：YYYY-MM-DD HH:mm
- 测试结果：通过 / 失败

## 测试结果汇总
| 测试项 | 状态 | 备注 |
| --- | --- | --- |
| TC-001 | PASS | |

## 发现的问题
| 优先级 | 问题描述 | 状态 |
| --- | --- | --- |
| HIGH | 描述 | OPEN |

## 结论
[是否可交付]
```
